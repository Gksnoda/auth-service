# Auth Service

## Introduction

This is a personal learning project where the user is building an authentication provider system (similar to Entra, Cognito, etc.) to deepen their Software Engineering skills.

## Rules

### Respect the User's Learning Process

- Only do what is explicitly requested. Do NOT add extra features, improvements, or suggestions beyond the scope of what was asked.
- The user wants to learn the logic and architecture themselves. Code implementation is fine — unsolicited design decisions are not.

### Language

- All code, comments, variable names, commit messages, documentation, and any text within the codebase MUST be written in English.
- The user may write messages in Portuguese. Respond in the same language the user uses.
- When the user writes in English, you MUST point out any grammar, spelling, or phrasing mistakes they make, as they are also learning English.
